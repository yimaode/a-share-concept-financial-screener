# Security Policy

## Supported version

安全修复优先应用于默认分支的最新版本。

## 报告安全问题

请不要在公开 Issue 中粘贴 API Key、Cookie、券商凭证、数据库密码、个人自选股或包含个人信息的财报处理结果。创建 GitHub 仓库后，请通过仓库的 **Security → Report a vulnerability** 私下报告。

报告应包括影响范围、复现步骤、受影响版本和建议修复方式，但不要包含真实生产凭证。

## 凭证原则

- 本项目默认数据链路不需要 API Key。
- 新数据源的凭证必须通过环境变量或本地密钥管理器注入。
- 禁止把凭证、Cookie、券商会话或数据库连接串写进代码、测试夹具、Notebook、日志和截图。
- 如果密钥曾进入 Git 历史，应立即吊销或轮换；仅删除当前文件不够。

## 投资数据安全

股票清单、持仓、自选股和研究笔记可能暴露个人投资策略。它们已被默认 `.gitignore` 排除，维护者仍应在提交前人工检查 `git diff --staged`。
